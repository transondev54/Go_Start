## 🔑 KEY CONCEPTS

### Những điều phải nhớ:

1. **Go là compiled** - Code phải được build trước khi chạy
2. **Go là strongly typed** - Phải chỉ rõ type
3. **Error handling là explicit** - Không exception, mà return error
4. **Simplicity > Features** - Go thiết kế để đơn giản
5. **Composition > Inheritance** - Dùng embedding thay class

## 💡 TIPS FOR SUCCESS

### ✅ Tips 1: Gõ tay tất cả code

Đừng copy-paste. Gõ tay giúp bạn:

- Nhớ cú pháp hơn
- Hiểu code hơn
- Tập muscle memory

### ✅ Tips 2: Experiment

Sau mỗi code example, hãy:

- Thay đổi giá trị
- Xem output thay đổi thế nào
- Thử break code (sẽ học từ errors)

### ✅ Tips 3: Read Error Messages

Go compiler messages rất rõ ràng:

- "undefined" → quên import
- "not enough arguments" → thiếu tham số
- "type mismatch" → type sai

### ✅ Tips 4: Take Breaks

- 2 giờ làm → 15 phút break
- Giữ tập trung
- Uống nước, đi bộ

### ✅ Tips 5: Ghi chép

- Ghi lại concepts khó
- Ghi lại mistakes
- Ghi lại questions → hỏi ở Lesson 8

## 📋 QUICK REFERENCE

### Must-know commands:

```bash
go version           # Check Go version
go mod init <name>   # Initialize module
go run main.go       # Compile & run
go build             # Build executable
go fmt               # Format code
go vet               # Check for errors
```

### Must-know packages:

```
fmt      - Formatting & printing
math     - Math functions
strings  - String manipulation
math/rand - Random numbers
time     - Date/time
errors   - Error creation
```

### Must-know patterns:

```go
// Error checking
if err != nil {
    return err
}

// Range loop
for i, v := range slice { }

// Type assertion
value, ok := interface{}.(Type)

// Defer cleanup
defer file.Close()
```
